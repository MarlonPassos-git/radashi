### Usage

Creates a function that always returns the same value, regardless of any arguments passed to it.

```ts
import * as _ from 'radashi'

const alwaysTrue = _.always(true)

alwaysTrue() // true
alwaysTrue(1, 2, 3) // true
```

### Use cases

You can avoid using `always` if the value is a primitive (use `() => true` instead), but it can be useful if you need a function that always returns the same object reference, or if you want to memoize a calculation across multiple calls.

```ts
// Not memoized
() => someCalculation()
// Memoized
_.always(someCalculation())

// Not same object
() => ({ a: 1, b: 2 })
// Same object
_.always({ a: 1, b: 2 })
```

### Search terms

- Often called `constant`, `alwaysReturn`, or `fixedValue`.

### Popular use cases

- Creating a function that consistently returns a predefined value regardless of input, useful for testing purposes or simplifying logic.
- Defining placeholder functions in place of complex logic during development or for mock implementations in unit testing.
- Caching a computed result that can be reused across different parts of an application, ensuring consistency and avoiding recalculation.
